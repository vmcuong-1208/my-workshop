# Build an AI Personal Learning Platform on AWS

In this workshop, you will build a serverless learning journal application that allows users to sign in, create study logs, upload images, and generate AI-powered summaries, reports, quizzes, search results, and analytics.

The implementation follows a practical AWS architecture with:

- **Amazon Cognito** for authentication and Hosted UI sign-in.
- **Amazon API Gateway + AWS Lambda** for protected API routes and business logic.
- **Amazon DynamoDB** for journal, report, quiz, and attempt data.
- **Amazon S3** for image storage using presigned URLs.
- **Amazon SQS** and a Lambda worker for asynchronous AI jobs.
- **Groq API** for fast AI generation.
- **CloudWatch, SNS, AWS Budgets, CloudFront, and WAF** for monitoring, alerts, and security.

## Workshop flow

1. **Identity and API foundation** – configure Cognito, create the main Lambda API, and protect routes with JWT authorizers.
2. **Storage and data layer** – create DynamoDB tables, S3 buckets, and journal routes.
3. **Asynchronous AI processing** – create SQS queues, a Lambda worker, and AI report/quiz routes.
4. **Search, deployment, and operations** – add search and analytics handlers, deploy the frontend with Amplify, and configure cost and security guardrails.
