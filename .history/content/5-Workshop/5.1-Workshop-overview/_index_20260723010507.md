#### Architecture flow

This workshop follows the architecture below. The user accesses the SPA through CloudFront. AWS WAF protects the public edge, Amplify hosts the frontend, and Cognito authenticates users before the frontend calls protected API Gateway routes.

![architecture](/images/5-Workshop/5.1-Workshop-overview/architecture.png)

#### Request path

1. User opens the frontend application.
2. AWS WAF filters unwanted traffic before it reaches CloudFront.
3. CloudFront serves SPA assets from Amplify.
4. Cognito authenticates the user and returns JWT tokens.
5. The SPA calls API Gateway with the access token.
6. API Gateway forwards requests to the main Lambda API.
7. Lambda performs CRUD operations on DynamoDB.
8. Lambda creates presigned S3 URLs for image upload and download.
9. For long-running AI features, Lambda sends jobs to SQS.
10. Failed jobs are routed to a DLQ for troubleshooting.
11. SQS triggers the Lambda AI Worker.
12. The worker calls Groq for prompt completion.
13. The worker saves AI reports, quiz data, and summaries in DynamoDB.
14. CloudWatch, SNS, and AWS Budgets provide logs, alerts, and cost guardrails.

#### Main components

- **Amazon Cognito** manages sign-up, sign-in, Google federation, and JWT-based authorization.
- **API Gateway + Lambda API** expose `/health`, `/me`, `/logs`, image upload, AI report, quiz, search, and analytics routes.
- **DynamoDB** stores journal logs, AI reports, quizzes, quiz attempts, and analytics-friendly data.
- **S3** stores uploaded learning images while access is controlled through presigned URLs.
- **SQS + Lambda Worker** decouple AI generation from user-facing API latency.
- **Groq API** provides fast Llama model completion for summaries, reports, and quizzes.
