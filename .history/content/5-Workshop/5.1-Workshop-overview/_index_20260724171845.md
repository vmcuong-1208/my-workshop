## Architecture flow

This workshop builds an AI-powered learning journal application on AWS. The user accesses the frontend through CloudFront, AWS WAF protects the public edge, Amplify hosts the web app, and Cognito authenticates the user before the frontend calls protected API Gateway routes.

![architecture](../images/5-Workshop/5.1-Workshop-overview/architecture.png)

## Request path

1. The user opens the frontend application.
2. AWS WAF filters unwanted traffic before it reaches CloudFront.
3. CloudFront serves the SPA assets from Amplify.
4. Cognito authenticates the user and returns JWT tokens.
5. The SPA calls API Gateway with the access token.
6. API Gateway forwards requests to the main Lambda API.
7. Lambda stores or retrieves data from DynamoDB.
8. Lambda creates presigned S3 URLs for image upload and download.
9. Long-running AI jobs are sent to SQS for asynchronous processing.
10. Failed jobs are routed to a DLQ for troubleshooting.
11. SQS triggers the Lambda AI Worker.
12. The worker calls Groq for AI completion.
13. The worker saves reports, quizzes, and summaries in DynamoDB.
14. CloudWatch, SNS, and AWS Budgets provide logs, alerts, and cost guardrails.

## Main components

- **Amazon Cognito** handles sign-up, sign-in, Google federation, and JWT-based authorization.
- **API Gateway + Lambda API** expose routes for health checks, profile info, journal CRUD, image upload, AI reports, quizzes, search, and analytics.
- **DynamoDB** stores journal logs, AI reports, quizzes, quiz attempts, and analytics-friendly data.
- **S3** stores uploaded learning images while access is controlled through presigned URLs.
- **SQS + Lambda Worker** decouple AI generation from user-facing API latency.
- **Groq API** provides fast Llama model completion for reports and quizzes.
