      <h1>Workshop overview</h1>
      <h4 id=architecture-flow>Architecture flow</h4>
      <p>This workshop follows the architecture below. The user accesses the SPA
        through CloudFront. AWS WAF protects the public edge, Amplify hosts the
        frontend, and Cognito authenticates users before the frontend calls
        protected API Gateway routes.</p>
      <p><img src=/images/5-Workshop/5.1-Workshop-overview/architecture.png
          alt=architecture></p>
      <h4 id=request-path>Request path</h4>
      <ol>
        <li>User opens the frontend application.</li>
        <li>AWS WAF filters unwanted traffic before it reaches CloudFront.</li>
        <li>CloudFront serves SPA assets from Amplify.</li>
        <li>Cognito authenticates the user and returns JWT tokens.</li>
        <li>The SPA calls API Gateway with the access token.</li>
        <li>API Gateway forwards requests to the main Lambda API.</li>
        <li>Lambda performs CRUD operations on DynamoDB.</li>
        <li>Lambda creates presigned S3 URLs for image upload and download.</li>
        <li>For long-running AI features, Lambda sends jobs to SQS.</li>
        <li>Failed jobs are routed to a DLQ for troubleshooting.</li>
        <li>SQS triggers the Lambda AI Worker.</li>
        <li>The worker calls Groq for prompt completion.</li>
        <li>The worker saves AI reports, quiz data, and summaries in DynamoDB.
        </li>
        <li>CloudWatch, SNS, and AWS Budgets provide logs, alerts, and cost
          guardrails.</li>
      </ol>
      <h4 id=main-components>Main components</h4>
      <ul>
        <li><strong>Amazon Cognito</strong> manages sign-up, sign-in, Google
          federation, and JWT-based authorization.</li>
        <li><strong>API Gateway + Lambda API</strong> expose
          <code>/health</code>, <code>/me</code>, <code>/logs</code>, image
          upload, AI report, quiz, search, and analytics routes.</li>
        <li><strong>DynamoDB</strong> stores journal logs, AI reports, quizzes,
          quiz attempts, and analytics-friendly data.</li>
        <li><strong>S3</strong> stores uploaded learning images while access is
          controlled through presigned URLs.</li>
        <li><strong>SQS + Lambda Worker</strong> decouple AI generation from
          user-facing API latency.</li>
        <li><strong>Groq API</strong> provides fast Llama model completion for
          summaries, reports, and quizzes.</li>
      </ul>
