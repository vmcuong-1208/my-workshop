# Restrict S3 access with an endpoint policy

When you create a gateway or interface endpoint for Amazon S3, you can attach an endpoint policy to limit which buckets or actions are allowed through that private path. This is a useful pattern when you want to keep private access narrow and avoid accidental data exposure.

## Why this matters

A default endpoint policy often allows broad access. By adding a custom policy, you can ensure that only the intended bucket is reachable through the endpoint and that other buckets remain protected.

## What you will do

1. Create or reuse a second S3 bucket for testing.
2. Update the VPC endpoint policy so that only that bucket is allowed.
3. Verify that access to the allowed bucket succeeds while access to the original bucket is denied.

This bonus lab demonstrates how to turn a general private path into a controlled, least-privilege access pattern.
