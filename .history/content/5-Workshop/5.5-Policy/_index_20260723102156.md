# Security, monitoring, and cost guardrails

## AWS Budgets

Create a cost budget so you receive alerts before the workshop exceeds the expected spend.

1. Open **AWS Budgets**.
2. Choose **Create budget**.
3. Select **Cost budget**.
4. Set a small monthly budget, for example USD 5 or USD 10.
5. Add email alerts at 50%, 80%, and 100%.
6. Create the budget.

## API Gateway throttling

Throttle the API to reduce accidental or abusive traffic:

```text
Rate limit: 5-10 requests per second for the workshop
Burst limit: 10-20 requests
```

Apply stricter limits to AI routes such as `/ai/reports` and `/quiz` because they can trigger downstream Groq usage.

## Lambda limits

Configure each Lambda function deliberately:

- Main API Lambda: short timeout, for example 10-15 seconds.
- AI Worker Lambda: longer timeout, for example 30-60 seconds.
- Reserved concurrency on AI Worker: small value such as 2-5.

## SQS DLQ

Make sure `ai-report-jobs` and `quiz-jobs` have a DLQ configured. Use the DLQ to inspect failed payloads and avoid silently losing AI requests.

## CloudWatch and SNS

1. Check CloudWatch Logs for each Lambda function.
2. Create metric filters or alarms for Lambda errors, throttles, and duration.
3. Send alarm notifications to an SNS topic.
4. Subscribe your email to the SNS topic and confirm the subscription.

## S3 and DynamoDB cost controls

- Keep S3 lifecycle rules for uploaded images if the workshop runs for many days.
- Use DynamoDB default/on-demand capacity during development.
- Delete unused test data after validation.
- Keep S3 Block Public Access enabled.

## WAF protection

For the frontend edge, attach WAF rules to block common bad requests and rate-limit repeated traffic. Start with AWS managed rule groups and tune based on sampled requests.
