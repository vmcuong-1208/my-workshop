## Required accounts and tools

Prepare the following before starting:

- An AWS account with permission to create **Cognito**, **Lambda**, **API Gateway**, **DynamoDB**, **S3**, **SQS**, **CloudWatch**, **SNS**, **Budgets**, **Amplify**, **CloudFront**, and **WAF** resources.
- A frontend repository, for example a React/Vite SPA, pushed to GitHub so Amplify can deploy it.
- A Google Cloud project if you want Cognito Hosted UI to support Google sign-in.
- A Groq account and API key for AI model completion.
- A local development URL such as `http://localhost:5173` for frontend testing.

## Recommended naming convention

Use consistent names to keep the workshop easier to follow:

- Cognito user pool: `AI-Personal-Learning-UserPool`
- Main Lambda API: `AI-Personal-Learning-Handle`
- Lambda AI Worker: `AI-Personal-Learning-Worker`
- HTTP API: `AI-Personal-Learning-API`
- S3 bucket: `learnflow-journal-images-<unique-suffix>`
- DynamoDB tables: `JournalLogs`, `AiReports`, `Quizzes`, `QuizAttempts`
- SQS queues: `ai-report-jobs`, `quiz-jobs`, and DLQs such as `ai-report-dlq` and `quiz-dlq`

## Security preparation

- Never hardcode the Groq API key in source code. Store it in Lambda environment variables or AWS Secrets Manager.
- Use Cognito JWT authorizers for all user-specific API routes.
- Keep S3 Block Public Access enabled and expose images only through presigned URLs.
- Start with least-privilege IAM policies where possible. Managed full-access policies are acceptable for a short workshop demo, but should be narrowed before production.

## Region

Use a single AWS Region consistently. In the sample workshop, the default region is **ap-southeast-1 (Singapore)** for Lambda, S3, SQS, and DynamoDB.
