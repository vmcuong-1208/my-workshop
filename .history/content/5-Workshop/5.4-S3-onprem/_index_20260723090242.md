#### Overview

# Add asynchronous AI processing

## Why asynchronous processing

AI report and quiz generation can take several seconds. If the frontend waits for the entire AI response inside one synchronous API call, users may see timeouts or slow screens. This section moves long-running work to SQS and a Lambda worker.

The main Lambda API only creates a pending job and returns immediately. The worker consumes the SQS message, calls Groq, and updates DynamoDB when the result is ready. The frontend then polls `GET /ai/reports/{id}` or `GET /quiz/{id}` until the status becomes `completed`.
