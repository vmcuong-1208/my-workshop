# Prepare AI data tables and queues

## AI DynamoDB tables

If you did not create these tables in the previous section, create them now:

```text
AiReports
Partition key: userId
Sort key: reportId

Quizzes
Partition key: userId
Sort key: quizId

QuizAttempts
Partition key: userId
Sort key: attemptId
```

Recommended status values:

```text
pending
processing
completed
failed
```

## SQS queues

Open **Amazon SQS** and create:

- `ai-report-jobs` for report generation jobs.
- `quiz-jobs` for quiz generation jobs.
- `ai-jobs-dlq` or separate DLQs for failed messages.

For each main queue:

1. Use **Standard queue**.
2. Configure a visibility timeout longer than the expected worker processing time.
3. Attach the DLQ with a maximum receive count such as `3`.

## IAM permissions

The main Lambda API needs permission to send messages to the two main queues. The worker needs permission to:

- Receive and delete messages from SQS.
- Read journal data from DynamoDB.
- Write AI results to `AiReports`, `Quizzes`, and `QuizAttempts`.
- Write logs to CloudWatch.
- Read the Groq key from environment variables or Secrets Manager.
