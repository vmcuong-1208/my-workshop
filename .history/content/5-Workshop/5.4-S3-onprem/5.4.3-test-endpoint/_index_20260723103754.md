# Connect API routes to asynchronous jobs

## Update the main Lambda API

The main Lambda API becomes the job coordinator. It should:

1. Receive a request from the frontend.
2. Create a DynamoDB record with `status: "pending"`.
3. Send a message to the correct SQS queue.
4. Return the job ID immediately to the frontend.
5. Support polling routes so the frontend can check job status.

## Add AI report routes

Create and protect these API Gateway routes:

- `POST /ai/reports`
- `GET /ai/reports`
- `GET /ai/reports/{id}`

`POST /ai/reports` creates a report item in `AiReports` and sends a message to `ai-report-jobs`. The message should include:

```json
{
  "reportId": "rep_...",
  "userId": "cognito-user-sub",
  "type": "weekly",
  "startDate": "2026-07-01",
  "endDate": "2026-07-07"
}
```

## Add quiz routes

Create and protect these API Gateway routes:

- `GET /quiz`
- `POST /quiz`
- `GET /quiz/{id}`
- `GET /quiz/{id}/attempts`
- `POST /quiz/{id}/attempts`

`POST /quiz` creates a quiz item in `Quizzes` and sends a message to `quiz-jobs`. Include the topic, selected tags, date range, question count, and difficulty.

## Polling flow

The frontend should:

1. Submit the AI request.
2. Store the returned `reportId` or `quizId`.
3. Poll the related `GET` route every few seconds.
4. Render a loading state while the status is `pending` or `processing`.
5. Render the result when the status is `completed`.
6. Show a retry message if the status is `failed`.
