# Kết nối các tuyến API với các tác vụ bất đồng bộ

## Cập nhật Lambda API chính

Lambda API chính sẽ trở thành bộ điều phối tác vụ. Nó cần:

1. Nhận một yêu cầu từ frontend.
2. Tạo một bản ghi trong DynamoDB với `status: "pending"`.
3. Gửi một thông điệp đến hàng đợi SQS phù hợp.
4. Trả về mã định danh tác vụ ngay lập tức cho frontend.
5. Hỗ trợ các tuyến polling để frontend có thể kiểm tra trạng thái tác vụ.

## Thêm các tuyến báo cáo AI

Tạo và bảo vệ các tuyến API Gateway sau:

- `POST /ai/reports`
- `GET /ai/reports`
- `GET /ai/reports/{id}`

`POST /ai/reports` tạo một mục báo cáo trong `AiReports` và gửi một thông điệp đến `ai-report-jobs`. Thông điệp cần bao gồm:

```json
{
  "reportId": "rep_...",
  "userId": "cognito-user-sub",
  "type": "weekly",
  "startDate": "2026-07-01",
  "endDate": "2026-07-07"
}
```

## Thêm các tuyến bài kiểm tra

Tạo và bảo vệ các tuyến API Gateway sau:

- `GET /quiz`
- `POST /quiz`
- `GET /quiz/{id}`
- `GET /quiz/{id}/attempts`
- `POST /quiz/{id}/attempts`

`POST /quiz` tạo một mục bài kiểm tra trong `Quizzes` và gửi một thông điệp đến `quiz-jobs`. Hãy bao gồm chủ đề, các thẻ đã chọn, khoảng ngày, số lượng câu hỏi và độ khó.

## Luồng polling

Frontend cần:

1. Gửi yêu cầu AI.
2. Lưu `reportId` hoặc `quizId` được trả về.
3. Gọi tuyến `GET` liên quan theo chu kỳ mỗi vài giây.
4. Hiển thị trạng thái đang tải khi trạng thái là `pending` hoặc `processing`.
5. Hiển thị kết quả khi trạng thái là `completed`.
6. Hiển thị thông báo thử lại nếu trạng thái là `failed`.
