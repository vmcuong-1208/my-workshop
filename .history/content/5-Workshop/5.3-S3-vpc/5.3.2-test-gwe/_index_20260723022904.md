# Create DynamoDB, S3, and test the API

#### Create DynamoDB tables

Create the base journal table first:

```text
Table name: JournalLogs
Partition key: userId
Sort key: entryId
Table settings: Default settings
```

Then create AI-related tables that will be used in the next section:

```text
Table name: AiReports
Partition key: userId
Sort key: reportId

Table name: Quizzes
Partition key: userId
Sort key: quizId

Table name: QuizAttempts
Partition key: userId
Sort key: attemptId
```

#### Grant Lambda data permissions

Open the execution role of `AI-Personal-Learning-Handle` and attach permissions for:

- DynamoDB read and write access to the four tables.
- S3 read and write access to the image bucket.
- SQS send-message access to the AI queues.
- CloudWatch Logs permissions for debugging.

For a short workshop, managed policies can speed up the build. For production, replace them with resource-scoped policies.

#### Create the S3 image bucket

1. Open **Amazon S3** and choose **Create bucket**.
2. Bucket name: `learnflow-journal-images-<unique-suffix>`.
3. Region: same Region as the Lambda API.
4. Keep **Block Public Access** enabled.
5. Create the bucket.

Configure bucket CORS so the frontend can upload through presigned URLs:

```json
[
  {
    "AllowedHeaders": ["*"],
    "AllowedMethods": ["GET", "PUT", "POST"],
    "AllowedOrigins": ["http://localhost:5173"],
    "ExposeHeaders": ["ETag"],
    "MaxAgeSeconds": 3000
  }
]
```

#### Add routes for journal and image upload

In API Gateway, create and attach the Lambda integration for:

```text
GET /logs
POST /logs
GET /logs/{id}
PUT /logs/{id}
DELETE /logs/{id}
POST /logs/{id}/images/presigned-url
```

Attach `CognitoAuth` to every route above.

#### Test end-to-end

1. Call `GET /health` from the browser. A successful response returns JSON similar to `{"ok":true}`.
2. Sign in through Cognito Hosted UI.
3. Call `GET /me` with the JWT token and verify that the API returns user information.
4. Create a journal log through `POST /logs`.
5. Request a presigned URL, upload an image to S3, and save the image key in the journal log.
6. Reload the frontend and verify that journal data and images are loaded for the signed-in user only.
