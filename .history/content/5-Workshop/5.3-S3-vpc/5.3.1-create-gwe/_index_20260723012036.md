# Configure Cognito and API Gateway

#### Create a Cognito user pool

1. Open **Amazon Cognito** and choose **User pools**.
2. Choose **Create user pool**.
3. In **Define your application**, choose **Single-page application (SPA)**.
4. Under **Options for sign-in identifiers**, choose **Email**.
5. Under **Required attributes for sign-up**, choose **email** and **name**.
6. Create the user directory.
7. Update the password policy to require at least 10 characters, uppercase letters, and numbers.

#### Configure Hosted UI and Google sign-in

1. Create a Cognito domain for Hosted UI.
2. In Google Cloud, configure the OAuth consent screen.
3. Create an OAuth Client ID for the Cognito Hosted UI.
4. Add the Google client ID and secret to Cognito.
5. Enable Google as an identity provider in Hosted UI.
6. Configure callback and sign-out URLs:

```text
Allowed callback URL: http://localhost:5173/auth/google/callback
Allowed sign-out URL: http://localhost:5173
```

#### Create the main Lambda API

1. Open **AWS Lambda** and choose **Create function**.
2. Choose **Author from scratch**.
3. Function name: `AI-Personal-Learning-Handle`.
4. Runtime: Node.js 22.x or the latest supported Node.js runtime available in your account.
5. Create the function and deploy the handler code.
