Công bố công cụ Tìm kiếm web trên Amazon Bedrock AgentCore dành cho truy xuất web theo hướng tác tử

Ngày 16/06/2026, AWS công bố Web Search trên Amazon Bedrock AgentCore – một công cụ tìm kiếm web “fully managed” được thiết kế riêng cho AI agents, giúp tác tử có thể truy xuất kiến thức web mới nhất, trả lời có trích dẫn nguồn, đồng thời không để dữ liệu egress (không “thoát” ra khỏi môi trường AWS bảo mật của khách hàng). Đây là một mảnh ghép rất đáng chú ý nếu bạn đang build agent theo hướng production: cần cập nhật theo thời sự, cần dẫn chứng, và cần tuân thủ bảo mật/doanh nghiệp.

Bài này mình sẽ phân tích nội dung chính trong tài liệu/tin tức và tổng hợp lại theo góc nhìn của một người học và chia sẻ cho cộng đồng.

1) Vấn đề cũ của AI agents: “biết” nhưng không “cập nhật”

LLM càng mạnh thì kỳ vọng người dùng càng cao: hỏi là phải ra đúng, mới, có căn cứ. Nhưng mô hình học từ dữ liệu quá khứ; khi gặp câu hỏi kiểu:

- “Tuần này AWS vừa ra tính năng gì liên quan AgentCore?”
- “Giá của Web Search tính theo gì?”
- “Thông tin nào là chính thức và có nguồn?”
…thì tác tử buộc phải đi ra ngoài mô hình để truy xuất web. Và đây là lúc các team hay gặp 3 bài toán đau đầu:
- Tích hợp nhà cung cấp search bên ngoài (API khác hệ, auth khác hệ).
- Tự điều phối (orchestration) luồng gọi tool – xử lý retries, rate limit, ranking, parsing snippet…
- Bảo mật & tuân thủ: truy vấn và dữ liệu có thể bị gửi ra ngoài hạ tầng kiểm soát của doanh nghiệp.

AWS định vị Web Search trên AgentCore là câu trả lời để “gỡ” bớt những phần nặng nhọc không tạo khác biệt, để dev tập trung vào logic agent.

2) Web Search trên AgentCore là gì?

Web Search là một công cụ được quản lý toàn phần, cho phép tác tử:

- gửi truy vấn ngôn ngữ tự nhiên

- nhận về kết quả xếp hạng gồm:

+ đoạn trích (snippets) ngắn, liên quan
+ URL nguồn
+ tiêu đề
+ ngày xuất bản
+ từ đó LLM suy luận và tạo câu trả lời có nền tảng (grounded)

Điểm nhấn lớn nhất: AWS nhấn mạnh cơ chế “zero data egress” — dữ liệu vẫn nằm trong môi trường AWS được bảo mật, không phải “đẩy prompt” sang một search API bên thứ ba.


3) “Khác gì so với search thường?”: Multi-source grounding + Knowledge Graph

Thay vì chỉ trả về danh sách link kiểu web search truyền thống, công cụ này được nói là xây dựng trên:

- chỉ mục web do Amazon vận hành (Amazon’s web index)

- kết hợp với dữ liệu đồ thị tri thức có cấu trúc (structured knowledge graph)

Nôm na: ngoài “link + snippet”, tác tử còn có thể tiếp cận thông tin thực thể (entity data) và các dữ kiện đã được xác minh (verified facts) từ Amazon Knowledge Graph. Hệ quả là:

- kết quả liên quan hơn

- câu trả lời chính xác hơn

- giảm rủi ro “lụm đại” một trang không uy tín và suy diễn sai

Với agentic workflows, chuyện “đúng nguồn – đúng sự thật” quan trọng hơn rất nhiều so với chỉ “tìm thấy link”.

4) Tối ưu cho agent: “high-value snippets” để tiết kiệm token

Một chi tiết mình thấy hay: Web Search được tối ưu để trả về đoạn trích giá trị cao nhằm tạo nhiều thông tin hữu ích trên mỗi token.

Ai từng build agent sẽ hiểu: token = tiền + latency. Nếu tool trả về nội dung dài, agent vừa tốn context vừa chậm. Web Search theo mô tả sẽ “đóng gói” thông tin dạng snippet để:

- model đọc nhanh

- suy luận nhanh

- vẫn có URL để trích dẫn/kiểm tra lại

5) Tích hợp bằng MCP qua AgentCore Gateway: đúng chất “agent platform”

Web Search được expose như một connector target tích hợp sẵn trong Bedrock AgentCore Gateway, thông qua Model Context Protocol (MCP).

Flow dễ hình dung:

	1. Bạn tạo AgentCore Gateway

	2. Chọn:

		+ Target protocol: MCP

		+ Target type: Connectors

		+ Tool: Web Search (preconfigured)

	3. Agent gọi tool bằng query tự nhiên

	4. Tool trả về ranked results + snippets + metadata

	5. Agent tổng hợp thành câu trả lời grounded

AWS cũng nhắc có thể test/debug thông qua MCP Inspector (một công cụ tương tác để chạy thử tool, rất tiện cho dev).

6) Nó nằm trong bức tranh AgentCore lớn hơn như thế nào?

Trong file bạn đưa có một phần tổng hợp rất “dễ nuốt” về AgentCore như một nền tảng fully managed cho AI agents, với nhiều mảnh ghép:

- Harness: vòng lặp agent được quản lý sẵn (memory, tools, môi trường cô lập)

- Runtime: nơi host agent và MCP servers an toàn

- Memory: ngắn hạn + dài hạn

- Gateway: kết nối tools (API/Lambda/MCP…)

- Identity: quản lý danh tính/quyền

- Browser: trình duyệt cloud an toàn cho automation

- Code Interpreter: chạy code trong sandbox

- Observability: tracing, debug, monitoring

Nhìn vào hệ này, Web Search giống như một “tool-native capability” – thêm trực tiếp vào hệ toolchain của agent thay vì bạn tự đi tích hợp bên ngoài.

7) Khía cạnh enterprise: governance và compliance

AWS đánh vào nỗi đau enterprise rất rõ:

- không gửi prompt và retrieval queries ra ngoài nhà cung cấp search third-party

- vẫn đáp ứng chính sách quản trị (governance policies)

- dễ kiểm soát trong môi trường AWS

Với các công ty có yêu cầu compliance cao, đây là khác biệt lớn giữa “demo chạy được” và “production deploy được”.

Những đoạn này giúp mình thấy AWS đang nhắm vào use case rất thực tế: grounded content + governed environment.

9) Availability & Pricing: thông tin quan trọng cho người muốn thử ngay

- Region GA: US East (N. Virginia)

- Pricing: $7 / 1,000 queries

- Có nhắc Free Tier credits lên đến $200 cho khách hàng AWS mới (tùy điều kiện)

Giá theo query là mô hình dễ dự trù khi bạn thiết kế agent: chỉ cần kiểm soát khi nào agent thật sự cần search.

10) Góc nhìn cá nhân: Ai nên quan tâm nhất?

Nếu bạn thuộc một trong các nhóm sau, Web Search trên AgentCore đáng để bookmark:

	  1. Build agent hỏi-đáp theo sự kiện mới (tin tức, cập nhật sản phẩm, chính sách…)

	  2. Agent cần “có dẫn chứng” (support, compliance, research assistant)

	  3. Team enterprise cần security/governance, không muốn prompt ra ngoài

	  4. Dev muốn tiết kiệm thời gian tích hợp: không muốn tự ráp search + auth + orchestration
