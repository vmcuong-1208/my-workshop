## Announcing Bedrock AgentCore Web Search for Agent-Oriented Retrieval

Amazon Bedrock AgentCore now supports web search as a retrieval source for agents. This update allows agents to use public web content as part of their knowledge base, making them more capable of answering questions that depend on up-to-date or external information.

### What is AgentCore?
AgentCore is a framework for building reasoning agents on Amazon Bedrock. It combines retrieval, memory, and planning, so an agent can search for relevant information, reason about it, and respond in a useful way.

### What does web search add?
With web search support, AgentCore agents can query the public web in real time, retrieve relevant pages, and incorporate those results into the agent’s retrieval pipeline. This is especially useful when the answer depends on recent events, news, or external documentation.

### Why this matters
Previously, agents relied mainly on internal sources such as local files or RAG datasets. Web search extends the agent’s knowledge scope dynamically, so it can access fresh content without requiring manual dataset updates.

### How it works
When a user asks a question, AgentCore sends the query to a search provider. The returned web results are converted into passages and used alongside other retrieval sources to support the agent’s answer.

### Who benefits most?
This feature is ideal for developers building agents that need:
- real-time information
- broad coverage beyond static datasets
- better handling of general knowledge queries
- stronger fact-finding capabilities for support or research workflows

### What’s next?
Web search support is a strong step toward more capable retrieval agents. As Bedrock AgentCore evolves, it will be able to combine web search with local content, structured RAG assets, and other Bedrock retrieval connectors, making agents more flexible and powerful.
